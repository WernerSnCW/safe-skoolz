# 070 — PRODUCT REVIEW PROMPT
Version: 1.1
Requires: A Step 1 current state scan run first. Do not run without a
CURRENT STATE OUTPUT — the review will be superficial.

---

This prompt assesses the tool as a product: how a user experiences it,
whether the organisation serves real workflows, and where friction exists.
It is not a technical health check (050) and not a strategic fit review (060).
Complete the template below exactly.

---

## CONSTRAINTS

- Assess what exists in CURRENT STATE OUTPUT. Do not design new pages or flows
  in this prompt — new flows belong under "Potential new features" only.
- For each user journey: reference at least one file, route, or component from
  the CURRENT STATE OUTPUT that implements it. If none exists: mark the journey
  as "Not currently implemented."
- Outputs from this prompt are advisory only. Any structural change proposed
  here must be pulled into a future Step 2 idea — it does not update CTX files
  or phase docs directly.
- Do not propose changes that contradict CTX_architecture.md without flagging
  them as requiring architectural review.

---

## INPUTS

**PRODUCT GOALS:**
Safeskoolz is a school safeguarding and wellbeing platform for Catalan/Spanish
schools. It supports incident reporting, protocol management (LOPIVI, Convivèxit,
Machista Violence), pupil wellbeing monitoring, and multi-role communication
across pupils, staff, parents, SENCO, coordinators, and PTA. Success means every
safeguarding workflow is complete end-to-end with no silent failures.

**PRIMARY USER:** Multiple roles — pupils (child-level), teachers, safeguarding
coordinators (senior staff), parents, SENCO, PTA members. Non-technical users.

**KNOWN PAIN POINTS:**
- DEMO_MODE / IS_DEMO desync: "Show me around" button renders in production
  but the underlying /auth/demo-login returns 403
- Password/PIN bypass is unconditional — any password works in all environments
- Inconsistent API call patterns (React Query hooks vs raw fetch) across pages
- /api/training/status route referenced in training.tsx but not mounted in
  routes/index.ts (unreachable)
- /api/learnings route referenced in learnings.tsx but not mounted (unreachable)
- Root-level artefact files (JSON/PNG) outside any organised directory
