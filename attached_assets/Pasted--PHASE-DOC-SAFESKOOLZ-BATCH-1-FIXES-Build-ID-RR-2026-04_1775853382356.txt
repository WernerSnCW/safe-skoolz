    write audit log entry

    audit log entry, then continue (do not fail the diary entry creation)

  AI-scanned. Manual review may be required."
- Notification trigger field: "diary_scan_skipped"
- Audit log eventType: "diary_scan_skipped"
- Audit log targetType: "diary_entry", targetId: the new diary entry id
- The coordinator to notify: look up the school's coordinator user the same
  way existing notification logic does in this file

**Files touched:** artifacts/api-server/src/routes/diary.ts
