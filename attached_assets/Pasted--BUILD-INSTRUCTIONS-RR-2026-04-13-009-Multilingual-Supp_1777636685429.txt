### Translation file rules

- Flat key structure only — no nested objects
- Keys are camelCase descriptive names
- `en/` values are the exact English strings from the source code
- `es/`, `nl/`, `fr/` values are machine translations
- Preserve `{{interpolation}}` placeholders exactly — do not translate
  text inside `{{ }}`
- Plural forms use i18next suffix convention:
  `"key_one": "{{count}} item"`, `"key_other": "{{count}} items"`
- Fallback: if a key is missing in es/nl/fr, English is shown automatically

### Namespace guide

| Namespace | Source files |
|---|---|
| common | Shared: Save, Cancel, Submit, Delete, Open, Closed, Pending, Loading... |
| nav | All sidebar + bottom nav labels from AppLayout.tsx |
| login | All strings in login.tsx |
| dashboard | All 4 dashboard components |
| incidents | report-incident.tsx, incidents/index.tsx, incidents/detail.tsx |
| protocols | protocols/index.tsx, protocols/detail.tsx, protocols/new.tsx |

## Step 4 — Wrap strings in all 24 components

For each component add at the top of the function body:
```typescript
const { t } = useTranslation("namespace");
```

Replace hardcoded strings:
```typescript
// Before
<h1>Report an Incident</h1>

// After
<h1>{t("incidents:reportTitle")}</h1>
```

Dynamic strings use interpolation:
```typescript
// Before
`Hi, ${user.firstName}!`

// After
t("dashboard:greeting", { name: user.firstName })

// en/dashboard.json:
{ "greeting": "Hi, {{name}}!" }
```

### Pluralisation — convert all 6 inline ternary patterns

| Before | After |
|---|---|
| `` `${data.total} incident${data.total !== 1 ? "s" : ""}` `` | `t("incidents:incidentCount", { count: data.total })` |
| `` `${data.total} report${...}` `` | `t("incidents:reportCount", { count: data.total })` |
| `` `${lockedPupils.length} pupil${...}` `` | `t("dashboard:lockedPupilCount", { count: lockedPupils.length })` |
| `` `${ptaContacts.length} PTA ${...}` `` | `t("dashboard:ptaMemberCount", { count: ptaContacts.length })` |
| Behaviour report count | `t("behaviour:reportCount", { count: N })` |
| `` `${pupil.failedLoginAttempts} failed attempt${...}` `` | `t("dashboard:failedAttemptCount", { count: pupil.failedLoginAttempts })` |

Plural JSON entries:
```json
{
  "incidentCount_one": "{{count}} incident",
  "incidentCount_other": "{{count}} incidents"
}
```

### DO NOT wrap in t() — leave as raw strings

- `"LOPIVI"`, `"Convivèxit"`, `"Machista Violence"`
- `"SENCO"`, `"GBV"`, `"PTA"`, `"safeskoolz"`
- Reference number format strings (`SS-YYYY-NNNN`, `PROT-YYYY-NNN`)
- Role values used in conditionals (`"coordinator"`, `"pupil"`, etc.)
- API error messages (`data.error`, `data.message` from fetch responses)
- User-authored content (`inc.description`, `msg.body`, `post.body`)
- Date values from API

### Role-dependent strings

Convert role-based ternaries to separate keys:
```typescript
// Before
isPupil ? "Are they a boy or a girl?" : "Gender"

// After
isPupil ? t("incidents:genderLabelPupil") : t("incidents:genderLabelStaff")

// en/incidents.json:
{
  "genderLabelPupil": "Are they a boy or a girl?",
  "genderLabelStaff": "Gender"
}
```

## Step 5 — Language picker on login screen

Add to `login.tsx` above the tab strip:

```typescript
import { useTranslation } from "react-i18next";
import i18n from "../lib/i18n";

// Inside component:
const handleLanguageChange = (code: string) => {
  i18n.changeLanguage(code);
  localStorage.setItem("safeskoolz_lang", code);
};

const currentLang = i18n.language;

// JSX:
<div className="flex gap-2 justify-center mb-4">
  {[
    { code: "en", label: "English" },
    { code: "es", label: "Español" },
    { code: "nl", label: "Nederlands" },
    { code: "fr", label: "Français" },
  ].map(({ code, label }) => (
    <button
      key={code}
      onClick={() => handleLanguageChange(code)}
      className={`text-sm px-3 py-1 rounded-full border transition-colors ${
        currentLang === code
          ? "bg-teal-600 text-white border-teal-600"
          : "text-gray-500 border-gray-300 hover:border-teal-400"
      }`}
    >
      {label}
    </button>
  ))}
</div>
```

The picker must be visible before authentication (it is on the login
screen, outside the auth context).
