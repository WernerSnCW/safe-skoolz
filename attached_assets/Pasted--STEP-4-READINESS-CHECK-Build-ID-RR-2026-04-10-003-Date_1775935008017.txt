## CHANGES MADE

1. (TYPE B) Atomic update gap — caught by code review and fixed during build.
   PATCH endpoint now uses scoped atomic WHERE clause preventing race
   conditions and cross-tenant data access. AC updated to reflect this.

2. (TYPE B) Route ordering issue — caught by Replit during implementation.
   GET /incidents/my-disclosures was initially placed after the /:id
   parameterised route (Express would have matched /my-disclosures as an
   :id value). Replit moved it above /:id. Phase doc constraint "static
   routes before parameterised routes" was correctly applied. No AC change
   needed — this is a constraint satisfaction.

3. (TYPE B) schoolId scoping added — Replit added tenant isolation to both
   new endpoints (scoped by schoolId from JWT). Phase doc did not explicitly
   require this but it is correct practice consistent with all other routes.

**Assumption-based changes:**
- disclosure_permissions schema confirmed: id, incidentId, requestedBy,
  parentId, status, requestedAt, respondedAt — matches phase doc assumption.
- enrichIncidents() confirmed to return raw Drizzle select results — new
  columns (acknowledgedAt, parentResponse) automatically included without
  modification needed.
- ParentDashboard.tsx used raw fetch() pattern — Replit matched this pattern
  for the new disclosures query.
