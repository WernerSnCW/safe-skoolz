lesson_progress (id uuid PK, school_id uuid FK schools, user_id uuid FK users, lesson_id uuid FK lessons, started_at timestamptz default now(), completed_at timestamptz nullable, quiz_score int nullable, reflection_text text nullable — SEE DECISION BELOW)

DECISION on reflection_text:
option A: store as plaintext, accessible only to the pupil who wrote it (never visible to coordinator). simplest.
option B: don't store at all. reflection is private; only completion is logged. safest from a safeguarding-disclosure-in-reflection-text perspective.
option C: store encrypted at rest, accessible by coordinator only if pupil explicitly flags for review.

recommendation: option B for v1. reflections are private to the pupil. if pupils need to flag something they wrote, they use the existing reporting/messages flow. document this choice in the schema file comments.

indexes on common query paths: school_id + key_stage on lessons, lesson_id on quizzes, user_id + lesson_id on progress, completed_at on progress.
